# Cycle 2026-06-06 — Retro

First real cycle on the meal-planning system. Friction + fixes for future runs.

## System bugs (block accuracy — fix first)

1. **`grocery.py` doesn't read `cycle.inventory_snapshot`** — only `deplete.py` does. Result: 20+ false-positive buys this cycle (chicken breast 25oz, bread, etc.) that I had to hand-strike when writing `2026-06-06-grocery.md`. Fix: have grocery subtract snapshot quantities the same way deplete does.

2. **`grocery.py` strict name+unit matching** — "Salt: need 1.50 tsp" doesn't match "Salt: 26 oz" in inventory. "Asiago: 1.70 oz" doesn't match "Asiago cheese 4 oz." Tons of spurious buys. Fix: name fuzzy match + unit conversion table (tsp/Tbsp/oz/lb).

3. **`defrost.py` assumes proteins live in freezer** — this cycle was all fresh-buy, but it still computed "move 2026-06-04" (date already past). Fix: skip defrost when the protein is on the grocery list (raw) for the same date as `cycle.date`.

4. **`cook_schedule.py` output is too coarse** — prints "perishable ingredients — cook first" 5x with no appliance routing, no time estimates, no day-of-week. I wrote `2026-06-06-cook-plan.md` by hand; it'll rot when the recipe DB changes.

## Schema upgrades (compound interest)

5. **Add `cook_method` + `cook_time` + `prep_time` + `appliance_required` to each recipe.** Enables a real `cook_schedule.py` (oven vs slow_cooker vs air_fryer vs Instant_Pot routing, parallel-block detection).

6. **Per-ingredient macro fields** — every new recipe this cycle needed hand-estimated macros (cal/protein/fat/carbs). Standard macro lookup → recipe-level totals computed deterministically. Removes a hand-estimation step + macro errors.

7. **Per-cycle store override** — Will swapped cottage cheese Walmart→Costco mid-planning. Edited `recipes.json` (source of truth). A `cycle.store_overrides: [{ingredient: "X", store: "Costco"}]` field keeps the recipe canonical while letting cycles flex.

8. **`product_sku` / `product_note` field** — the chipotle thigh went through 3 product variants (raw → pre-cooked grilled → pre-cooked diced). Free-text name field is too vague. A `product_note` field anchors which exact Costco SKU.

## New scripts (low-effort, high-value)

9. **`macro_summary.py`** — aggregates weekly cal/protein/fat/carbs per member across all selections. Would have shown earlier that the cycle was carb-heavy / chicken-heavy before we added #26 salmon + #27 soup. ~40 lines.

10. **`balance_check.py`** — surfaces "warning: 5/6 recipes share protein X" or "no fish/omega-3 this cycle." Catches the protein-monotony issue automatically (we caught it mid-conversation manually).

11. **`inventory_add.py` / `inventory_remove.py`** — Will edited `inventory.json` multiple times this cycle (added 16 items, removed 2 stale). CLI helpers prevent JSON typos and standardize names.
